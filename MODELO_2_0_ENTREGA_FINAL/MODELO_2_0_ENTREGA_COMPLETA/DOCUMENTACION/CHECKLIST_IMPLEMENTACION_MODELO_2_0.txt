RESUMEN EJECUTIVO DEL MODELO 2.0
═════════════════════════════════════════════════════════════════════════════════

El MODELO 2.0 es un SISTEMA DE EVALUACIÓN MULTIDIMENSIONAL que integra:

┌─────────────────────────────────────────────────────────────────────────────┐
│ 3 CAPAS DE ANÁLISIS:                                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│ 1. COGNITIVA (Cómo piensan/sienten los actores)                            │
│    └─ 8 dimensiones: IA, IAT, ICI, IVC, IME, IIN, IPRA, ICS               │
│    └─ 5-6 preguntas Likert por dimensión                                   │
│    └─ Rango: 0-100                                                         │
│                                                                             │
│ 2. OPERATIVA-MATERIAL (Qué infraestructura/recursos existen)               │
│    └─ Infraestructura física                                               │
│    └─ Capacidad técnica instalada                                          │
│    └─ Recursos disponibles                                                 │
│    └─ Cadenas productivas conectadas                                       │
│    └─ IOM índice: 0-100                                                    │
│                                                                             │
│ 3. TERRITORIAL (Contexto de pobreza, educación, empleo)                    │
│    └─ Indicadores DANE                                                     │
│    └─ Proxies administrativos                                              │
│    └─ Comparativo nacional                                                 │
│                                                                             │
├─────────────────────────────────────────────────────────────────────────────┤
│ ADEMÁS:                                                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│ 4. MEMORIA TEMPORAL (Cómo han cambiado las cosas en 12 meses)              │
│    └─ Series temporales mensuales                                          │
│    └─ Detección de shocks/eventos                                          │
│    └─ Análisis de tendencias                                               │
│    └─ Proyecciones a corto plazo                                           │
│                                                                             │
│ 5. PANEL 0 (5 actores + readiness global)                                  │
│    └─ Comunidad local                                                      │
│    └─ Gobierno local                                                       │
│    └─ Proveedores técnicos                                                 │
│    └─ Actores económicos                                                   │
│    └─ Actores externos                                                     │
│                                                                             │
│ 6. PUERTAS DE TRANSICIÓN (GATES)                                           │
│    └─ 3 estados: BLOCKED / CONDITIONAL / ENABLED                           │
│    └─ Criterios matemáticos claros                                         │
│    └─ Costo de transición calculado                                        │
│    └─ Recomendaciones específicas                                          │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

RESULTADO: MATRIZ DE 45 MICRO-REDES (5 actores × 3 fases × 3 subcapas)
CADA micro-red genera: 1 JSON estructurado con diagnóstico completo

DETALLES TÉCNICOS ESPECÍFICOS
═════════════════════════════════════════════════════════════════════════════════

REQUISITOS DE SOFTWARE:
  ├─ Python 3.9+
  ├─ Streamlit 1.28+
  ├─ Pandas 2.0+
  ├─ Numpy 1.24+
  ├─ Plotly 5.17+
  ├─ Openpyxl 3.1+
  └─ (Ver requirements.txt completo en repo)

REQUISITOS DE DATOS:
  ├─ Mínimo 60 encuestas para dimensiones cognitivas
  ├─ Datos DANE 2024 para contexto territorial
  ├─ Historial de 6 meses mínimo para memoria temporal
  ├─ Inventario físico completo de infraestructura
  └─ Capacidad técnica de 2-3 análisis simultáneos

INFRAESTRUCTURA RECOMENDADA:
  ├─ Servidor: 2 CPU, 4 GB RAM, 50 GB disco (mínimo)
  ├─ BD: PostgreSQL (opcional, para escalabilidad)
  ├─ Backup: Automático diario
  ├─ Certificado SSL: Obligatorio si es online
  └─ Ancho banda: 10 Mbps mínimo

MATRIZ DE DECISIONES CLAVE
═════════════════════════════════════════════════════════════════════════════════

DECISIÓN 1: ¿Empezar con muestra piloto o datos completos?
  ➜ RECOMENDACIÓN: Piloto (3 actores, 1 fase)
  ➜ VENTAJA: Rápido, aprendes el proceso
  ➜ TIEMPO: 2-3 semanas vs 6-8 semanas

DECISIÓN 2: ¿Encuesta propia o usar datos existentes?
  ➜ RECOMENDACIÓN: Ambos (triangulación)
  ➜ ENCUESTA: 60-100 personas (2-3 semanas)
  ➜ DATOS EXISTENTES: Proyectos previos, estudios locales

DECISIÓN 3: ¿Plataforma: Cloud vs Local vs Servidor?
  ➜ RECOMENDACIÓN: Depende de:
  ├─ Cloud: Si <100 usuarios/mes, datos públicos
  ├─ Local: Si datos sensibles, <20 usuarios
  ├─ Servidor: Si datos sensibles, >20 usuarios

DECISIÓN 4: ¿Cuántos actores incluir?
  ➜ RECOMENDACIÓN: Empezar con 3 (Comunidad, Gobierno, Técnicos)
  ➜ AGREGAR: Actores económicos + externos después
